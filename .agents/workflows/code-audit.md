Step 1: Execute analyze_component (for a specific file) or analyze_codebase_comprehensive (for a wider health check).

Step 2: Execute optimize_performance to identify and resolve render bottlenecks or memory leaks.

Step 3: Execute analyze_test_coverage to identify missing edge cases in the current testing strategy.

Step 4: Execute remediate_code or refactor_component to apply the recommended fixes automatically.

When warnings are given to you, consider their context/how they are used as well to determine if or how it needs to be fixed
